# CarrotBotJS

CarrotBotJS is a Discord bot built with Node.js, TypeScript, and `discord.js`. It powers a mix of fun utilities, admin helpers, scheduled events, and integrations with MongoDB and the Hypixel API.

It is a personal bot consisting of features meant for me and friends - but could be useful for others.
It has a Hypixel Paintball discord pinger when player counts reach a certain level.
It has a Hypixel Bedwars winstreak leaderboard system (for the Skillless Guild).

## Overview

- Bot commands organized by category (Admin, Fun, Skillless-Leaderboard)
- Interactive components via buttons and reactions
- Scheduled tasks (e.g., Walter Wednesday, Paintball player count)
- MongoDB persistence (via `mongoose`)
- External API integration (Hypixel)
- Clean event handling for Discord interactions and message reactions

## Getting Started

### Prerequisites

- Node.js and npm installed
- A Discord application with a bot token
- Optional: Access to a MongoDB instance

### Install

```powershell
# From the project root
npm install
```

### Build

Since the bot is written in TypeScript, you must bundle it before running:

```powershell
# Bundle TypeScript to JavaScript using tsup (outputs to dist/)
npm run build
```

### Run

```powershell
# Production: Run the compiled bot
npm start

# Development: Run with hot reload (no build needed)
npm run dev
```

## Configuration

The file `src/config.ts` holds static IDs and feature settings used across the bot (schedules, reactions, roles). Update the IDs to match your server and channels.
These are not pieces of sensitive information, and the current config is set up for my usecase.

- `paintball`: `guildId`, `channelId`, `roleName`, thresholds (`numToAlert`, `minutesBetweenAlerts`), data files
- `walterWednesday`: `guildId`, `channelId`, schedule `rule` (day/hour/minute/tz), `url`
- `userIds`, `bedwarsLeaderboard`: owner and role settings

Usage (relative path will vary by file location):

```ts
import config from "./config.js";
const guildId = config.walterWednesday.guildId;
```

## Environment Variables (.env)

Create a `.env` file in the project root. Example layout:

```env
# Discord bot token
token="aaaaaaaaa-11111-3333-444-2222222222"

# Comma-separated list of guild IDs (no spaces)
guildIds="guildId1,guildId2"

# MongoDB connection string
dbToken="mongodb+srv://bot:blah@blah-vlah.blah.mongodb.net/test?retryWrites=true&w=majority"

# Discord application client ID
clientId="clientId"

# Hypixel API key
hypixelApiKey="aaaaaaaaa-11111-3333-444-2222222222"
```

Notes:

- Do not commit your `.env` file.
- Ensure `guildIds` are comma-separated with no spaces.

## Deployment

### Important: TypeScript Migration

The bot is now written in TypeScript and **must be built before running**. When deploying to a server (e.g., Google Cloud VM):

1. **Upload your code** (zip or git clone)
2. **Install dependencies**: `npm install`
3. **Build the project**: `npm run build` (bundles TypeScript using tsup, creates `dist/` folder)
4. **Run the bot**: `npm start` (runs `node dist/bot.js`)

**What's different from before:**

- Old: Could run `node src/bot.js` directly
- Now: Must run `npm run build` first, then `npm start`

**Files to include in deployment:**

- All `src/` files (TypeScript source)
- `package.json` and `package-lock.json`
- `.env` file (create on server, never commit!)
- `tsconfig.json` and `tsup.config.ts` (build configuration)
- `dist/` folder is generated by build, don't need to upload it

### Setting up as a Linux Service

If running as a systemd service, make sure the service runs `npm start` after building, not `node src/bot.js`.

Example service file (`/etc/systemd/system/carrotbot.service`):

```ini
[Unit]
Description=CarrotBot Discord Bot
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/CarrotBotJS
ExecStart=/usr/bin/npm start
Restart=always

[Install]
WantedBy=multi-user.target
```

Make sure to run `npm run build` once after deployment and whenever you update the code.

## Project Structure

```
src/
 bot.ts                 # Bot entrypoint
 config.ts              # Configuration helpers
 types/                 # TypeScript type definitions
  bot.ts             # Command, Button, Event, API response types
 commands/              # Slash/message commands by category
 buttons/               # Button handlers (Fun, Skillless-Leaderboard)
 events/                # Discord event handlers (ready, interactions, reactions)
 functions/             # Shared utilities (db login, command/event routing, schedules)
 mongoEvents/           # MongoDB connection lifecycle handlers
 schedules/             # Scheduled jobs (Paintball, Walter Wednesday)
 schemas/               # Mongoose schemas (skillless-bwldb)
dist/                    # Compiled JavaScript output (generated by npm run build)
logs/                    # Runtime logs
```
